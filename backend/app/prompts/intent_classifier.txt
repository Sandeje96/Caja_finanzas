Analiza el siguiente mensaje del usuario y clasifica su intención.

INTENCIONES POSIBLES:
- REGISTER_EXPENSE: el usuario quiere registrar un gasto
- REGISTER_INCOME: el usuario quiere registrar un ingreso
- ATTACH_RECEIPT: el usuario está enviando o mencionando un comprobante/ticket/factura
- QUERY: el usuario hace una pregunta sobre sus finanzas
- UPDATE_TRANSACTION: el usuario quiere corregir o modificar un movimiento anterior
- HELP: el usuario necesita ayuda o no sabe qué puede hacer
- UNKNOWN: el mensaje no se puede clasificar con certeza

EJEMPLOS:
"gasté 15 lucas en combustible" → REGISTER_EXPENSE
"cobré 850 mil de sueldo" → REGISTER_INCOME
"te mando el ticket" → ATTACH_RECEIPT
"cuánto gasté este mes" → QUERY
"en realidad eran 20000, no 18000" → UPDATE_TRANSACTION
"qué puedo hacer acá" → HELP

Responde ÚNICAMENTE con un objeto JSON con esta estructura exacta:
{
  "intent": "REGISTER_EXPENSE",
  "confidence": 0.97,
  "entities": {
    "amount": 15000,
    "category": "Combustible",
    "description": "combustible",
    "date": null
  }
}

REGLAS:
- "confidence" debe ser un número entre 0 y 1
- Para extraer las "entities", respeta que "amount" sea numérico convertido (ej. 15 lucas = 15000, 850 mil = 850000, 2 palos = 2000000, medio palo = 500000).
- "category" debe ser el nombre de categoría más probable, o null
- "date" debe ser una fecha ISO 8601 si se menciona, o null para hoy
- Si la intención es QUERY, UPDATE_TRANSACTION, HELP o UNKNOWN, "entities" puede ser {}
- Responde SOLO con el JSON validado (sin bloques de código markdown ni texto adicional).
- FECHA DE HOY: {today}
